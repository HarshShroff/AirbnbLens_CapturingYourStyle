# 🦁 AirbnbLens Beast Engine

## 📖 Introduction
AirbnbLens is not just a standard search tool—it is a **Style-Discovery Engine**. Traditional search forces a user to know exactly what they are looking for (e.g., "3 bedroom, 2 bath in downtown Boston"). The **Beast Engine** solves the gap between what users *imagine* and what they can *describe*. 

By leveraging deep learning (ResNet152) and vector similarity search (ChromaDB), the Beast Engine allows users to find accommodations based purely on "vibe," aesthetic, or visual similarity. It fundamentally changes the dynamic of travel discovery from "Where do you want to stay?" to **"How do you want to feel?"**

---

## 🏗️ Architecture

The Beast Engine is built on a two-tier, high-performance architecture optimized for production-level scale tracking over **113,000 global listings**.

### 1. The Vector Brain (ChromaDB)
To handle the "curse of dimensionality," the database is split into two specialized collections:
*   **`airbnb_visuals`**: A high-precision collection containing **2048-dimensional embeddings** generated by the ResNet152 neural network. This collection is exclusively used for matching the visual styles of images uploaded by users against the pictures of Airbnb listings.
*   **`airbnb_metadata`**: A secondary collection containing **384-dimensional embeddings** representing the metadata, descriptions, and "vibes" of the accommodations.

### 2. The Cloud-Ready API (FastAPI)
The backend acts as a highly concurrent bridge between the user's natural language / image inputs and the ChromaDB brain. 
*   It loads the `ResNet152` Convolutional Neural Network into memory **once at startup**, stripping the top classification layer to use it purely for feature extraction.
*   By utilizing a local `PersistentClient` with SQLite, it maintains safe, fast, thread-ready access to the vector databases without the need for complex, bug-prone HTTP server middleware.

## ⚡ Current Capabilities (What is it doing right now?)

1.  **Multi-City Aesthetic Pre-fetching (The Cold Start)**
    *   Instead of an empty search screen, the app immediately hits the `/search/text` API with a query like *"Stunning layout in Asheville"* to pre-populate the UI with the top visually appealing listings in the currently selected region.
    
2.  **Natural Language Vibe Discovery**
    *   Users can type *"Cozy brick loft with plants"* or *"Dark academia writer's retreat."* The query is converted into a vector and searched against the 113,000 listings in milliseconds, mapping intent and sentiment rather than purely querying matching keywords.

3.  **Visual Lens Search (The Core Feature)**
    *   Users can upload a photo from their camera roll or Pinterest board. The API feeds the image through ResNet152 in real-time to generate a 2048-dimensional "Style Fingerprint." It then queries the DB to find the listings with the absolute closest mathematical aesthetic match.

4.  **Style Teleportation / Migration**
    *   *The "Beast" Feature:* If a user finds a visually stunning listing in Boston but actually needs to travel to Chicago, they can click a single button. The system grabs the ResNet vector of the Boston listing and searches the Chicago directory for an "Aesthetic Twin."

5.  **Dynamic Vibe Match Scoring**
    *   Instead of simple binary results, the frontend intelligently ranks the listings and displays a percentage "Match Score" using a dynamic gradient overlay to visually represent how close a listing is to the user's desired style.

## 🔮 Future Capabilities (What should it do next?)

1.  **LLM Integration (The OpenAI Expansion)**
    *   **Contextual Refinement:** Passing the user's natural language query to an LLM *before* searching to extract hyper-specific intent. (e.g., Transforming *"I want a romantic weekend"* into *"Search for: dim lighting, fireplace, secluded, luxury"*).
    *   **Review Summarization:** Asking an LLM to read through 50 user reviews of a matched listing and automatically generate a two-sentence summary of the *Aesthetic Experience*.

2.  **The "Host Competitor" Dashboard**
    *   A completely separate view designed for Airbnb Hosts. A host can input their own listing URL, and the Beast Engine will analyze the photos to identify their **"Style Competitors"** within a 5-mile radius, potentially even advising on what furniture or lighting to update to match current "trending" styles in that specific neighborhood.

3.  **Aesthetic Heatmapping (Geospatial Integration)**
    *   Merging the ChromaDB results with a frontend Map interface (e.g., Mapbox). Instead of just showing pins, the map could color-code neighborhoods by their dominant vibe (e.g., the "Minimalist" district vs. the "Rustic" district of a city).

4.  **Advanced User Profiling**
    *   Saving historical searches and teleportations to assemble a "Style Passport" for authenticated users. The moment they open the app, it instantly knows they prefer "Mid-Century Modern" and auto-filters global destinations to match.
